# ADR 0002: Фоновые сервисы (BackgroundService) для сбора данных с бирж

**Статус**: Принято  
**Дата**: 2026-06-14

## Контекст

Нужно периодически опрашивать API бирж (Binance, Bybit, OKX) для получения:
- Текущих ставок финансирования (каждые 10 секунд)
- Исторических данных (по событию изменения ставки/нового символа)

Варианты: Hangfire, Quartz, Hosted Services (IHostedService/BackgroundService), внешние cron-джобы.

## Решение

Используем **`BackgroundService` (ASP.NET Core Hosted Services)** — встроенные в .NET Generic Host.

Два сервиса:
1. `CurrentCollectionBackgroundService` — интервал 10 сек, параллельный опрос 3 бирж
2. `HistoricalCollectionBackgroundService` — потребляет очередь Redis, ограничивает параллелизм `SemaphoreSlim`

Запускаются автоматически при старте хоста, поддерживают graceful shutdown через `CancellationToken`.

## Последствия

✅ **Плюсы**:
- Нет внешних зависимостей (Hangfire, Quartz, Redis Queue workers)
- Простая отладка — это просто C# классы в том же процессе
- Единое логирование, DI, конфигурация
- Graceful shutdown из коробки

❌ **Минусы**:
- При масштабировании (несколько инстансов API) — дублирование сбора
- Нет встроенного ретрая/планирования (пишем сами)
- Нет дашборда мониторинга джобов

## Альтернативы

| Вариант | Причина отказа |
|---------|----------------|
| Hangfire | Overkill для 2 джобов, требует БД/Redis, лишний вес |
| Quartz.NET | Сложная конфигурация, избыточен для простых интервалов |
| Внешний cron / systemd | Разрыв деплоя, сложнее управлять средой |
| Temporal / Orkes | Слишком сложно для текущих нужд |

## Планы на будущее

При горизонтальном масштабировании API — вынести сбор в отдельный Worker Service / контейнер с лидер-элекшеном (Redis lock) или использовать Hangfire.
